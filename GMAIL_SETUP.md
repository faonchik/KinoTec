# Настройка Gmail для сброса пароля

## Шаг 1: Включение двухфакторной аутентификации

1. Перейдите в [настройки аккаунта Google](https://myaccount.google.com/)
2. Выберите **Безопасность**
3. Включите **Двухэтапную аутентификацию** (если еще не включена)

## Шаг 2: Создание пароля приложения

1. Перейдите в [настройки аккаунта Google](https://myaccount.google.com/)
2. Выберите **Безопасность**
3. Найдите раздел **Вход в аккаунт Google**
4. Нажмите **Пароли приложений** (App Passwords)
5. Выберите приложение: **Почта**
6. Выберите устройство: **Другое (название по вашему выбору)**, например "КиноТека"
7. Нажмите **Создать**
8. Скопируйте 16-значный пароль (без пробелов)

## Шаг 3: Настройка переменных окружения

Добавьте в файл `.env` или `.env.local`:

```env
# Gmail SMTP настройки
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=ваш-email@gmail.com
SMTP_PASS=ваш-16-значный-пароль-приложения
SMTP_FROM=ваш-email@gmail.com

# Или используйте альтернативные переменные
GMAIL_USER=ваш-email@gmail.com
GMAIL_APP_PASSWORD=ваш-16-значный-пароль-приложения
GMAIL_FROM=ваш-email@gmail.com
```

## Важно!

- **НЕ используйте** обычный пароль от Gmail аккаунта
- **Используйте ТОЛЬКО** пароль приложения (App Password)
- Пароль приложения состоит из 16 символов без пробелов
- Если вы не видите опцию "Пароли приложений", убедитесь, что включена двухфакторная аутентификация

## Проверка работы

После настройки попробуйте запросить сброс пароля на странице `/auth/forgot-password`. Если настройки верны, письмо должно прийти на указанный email.

## Безопасность

- Никогда не коммитьте файл `.env` в Git
- Храните пароль приложения в безопасном месте
- Если пароль приложения скомпрометирован, удалите его и создайте новый
